## 1. 引言

酒店管理系统面向课程实践中的前台、客房服务、管理和测试演示场景，目标是完成客房管理、预订、入住、退房、账单、清洁、客史和报表等核心业务闭环。系统由 Spring Boot 后端、MySQL 数据库和 uni-app 前端组成，支持网页端和 APP 端源码形态。

## 2. 用户角色

| 角色 | 主要目标 | 典型操作 |
| --- | --- | --- |
| 管理员 | 管理系统数据并查看经营状态 | 登录、查看房态、维护房间、查看报表 |
| 前台人员 | 处理预订、入住和退房 | 新增预订、办理入住、消费入账、退房结算 |
| 客房服务员 | 处理清洁任务并反馈房态 | 查看清洁任务、更新任务状态、完成检查 |
| 测试人员 | 验证业务闭环和交付质量 | 执行自动化测试、冒烟测试、缺陷记录 |

## 3. 功能需求

| 编号 | 模块 | 需求描述 | 验收要点 |
| --- | --- | --- | --- |
| FR-01 | 认证登录 | 用户可使用测试账号登录并获取 token | 返回用户身份、角色和可用于后续请求的 token |
| FR-02 | 房态管理 | 查询全部房间、按楼层/状态筛选、查看房间详情 | 房态枚举统一为 `vacant/occupied/reserved/dirty/cleaning/maintenance` |
| FR-03 | 预订管理 | 创建和取消预订，预订成功后占用对应房间 | 不允许无效日期或无可用房间的预订 |
| FR-04 | 入住登记 | 支持预订入住和 Walk-in 入住 | 生成入住记录、客史和账单，房间变为入住 |
| FR-05 | 退房结算 | 结清账单并释放入住记录 | 房间进入脏房状态并自动生成清洁任务 |
| FR-06 | 账单管理 | 查询账单、查看明细、追加服务消费 | 金额计算使用精确小数，服务费进入汇总 |
| FR-07 | 清洁管理 | 清洁任务从待处理到进行中再到完成 | 检查通过后房间恢复空净 |
| FR-08 | 客史管理 | 查询住客历史信息 | 退房和新增入住后数据可追溯 |
| FR-09 | 报表统计 | 展示入住率和营收统计 | 返回总房数、入住率、总营收和账单数 |
| FR-10 | 前端展示 | 网页端和 APP 端提供主要页面源码 | 页面路径、组件结构和图标资源完整 |

## 4. 非功能需求

| 类型 | 要求 |
| --- | --- |
| 可用性 | 后端支持本地 `java -jar` 启动，数据库脚本可重建测试数据 |
| 可维护性 | 后端采用 Controller、Service、Mapper、Entity、DTO 分层 |
| 可测试性 | 至少保留 Maven/JUnit 自动化测试和 API 冒烟测试记录 |
| 一致性 | API 返回统一响应结构，文档中的接口和枚举与代码一致 |
| 安全性 | 明确标记数据库密码、Token secret、前端地址硬编码风险 |

## 5. 范围边界

本次交付覆盖本地运行、接口验证和课程答辩所需文档。前端目录当前没有 `package.json`，因此不承诺命令行前端构建和浏览器 E2E 已完成。WebSocket 已有最小实现，但本次测试记录未覆盖真实客户端订阅验证。
