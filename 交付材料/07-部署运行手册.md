## 1. 环境要求

| 工具 | 版本建议 |
| --- | --- |
| JDK | 17 或更高 |
| Maven | 3.6 或更高 |
| MySQL | 8.0 |
| 前端工具 | HBuilderX / uni-app 相关环境 |

## 2. 初始化数据库

在 MySQL 客户端中按顺序执行：

```sql
source 后端/hrms-backend/src/main/resources/db/schema.sql;
source 后端/hrms-backend/src/main/resources/db/seed.sql;
```

如果使用图形化工具，先执行 `schema.sql`，再执行 `seed.sql`。

## 3. 后端配置

配置文件位置：

```text
后端/hrms-backend/src/main/resources/application.yml
```

默认端口为 `8080`。如果端口被占用，可启动时追加：

```powershell
--server.port=18080
```

如果本机 MySQL 密码不同，可启动时追加：

```powershell
--spring.datasource.password=<本机有效密码>
```

## 4. 后端启动

```powershell
cd "后端/hrms-backend"
mvn test
mvn package -DskipTests
java -Dfile.encoding=UTF-8 -jar "target/hrms-backend-0.1.0.jar"
```

## 5. 接口文档

- Swagger UI：`http://localhost:8080/swagger-ui.html`
- Knife4j：`http://localhost:8080/doc.html`
- 测试账号：`http://localhost:8080/api/auth/test-accounts`
- HTTP 样例：`后端/http/hrms-api.http`

## 6. 前端运行

网页端和 APP 端均为 HBuilderX/uni-app 工程。网页端可使用 HBuilderX 执行发行流程，生成静态资源后放入 Nginx 目录，参考 `后端/nginx/hrms-web.conf`。

## 7. 部署风险

- 默认数据库密码不一定适配所有机器。
- 前端网页端 API 地址当前写死为 `http://localhost:8080`。
- 当前前端目录缺少 CLI 构建入口，CI 构建需要后续补齐 `package.json`。
