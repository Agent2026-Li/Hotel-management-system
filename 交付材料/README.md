本目录用于课程提交和答辩审核，集中保存测试工程师与文档助理负责的材料。Markdown 文件是可追溯源文件，`酒店管理系统_全套软件工程文档.docx` 是统一排版后的 Word 成品，`酒店管理系统_答辩PPT初稿.pptx` 是答辩 PPT 初稿。

## 目录说明

| 文件 | 用途 |
| --- | --- |
| `01-需求规格说明书.md` | 系统背景、用户角色、功能需求和非功能需求 |
| `02-概要设计说明书.md` | 总体架构、模块划分、核心流程和部署视图 |
| `03-详细设计说明书.md` | 后端分层、前端页面、接口调用和异常处理细节 |
| `04-数据库设计说明书.md` | 核心表、字段职责、测试数据和约束说明 |
| `05-接口设计说明书.md` | 统一响应、鉴权、主要 API 和数据约定 |
| `06-用户手册.md` | 面向演示和验收的功能操作说明 |
| `07-部署运行手册.md` | 后端、数据库、前端和常见问题处理 |
| `08-软件测试报告.md` | 测试范围、执行记录、缺陷风险和补测建议 |
| `09-答辩材料清单.md` | 答辩 PPT、演示、文档和提交材料检查 |
| `10-演示视频脚本和录制清单.md` | 演示视频镜头顺序、旁白稿和录制检查项 |
| `11-格式排版查重自查表.md` | 统一格式、排版、原创性和引用自查 |

## 交付原则

- 文档统一使用 UTF-8 编码、Markdown 源文件和 Word 成品双轨维护。
- 所有结论来自本地仓库、运行测试和课程岗位文档，不编造未验证功能。
- 查重部分只做本地原创性和格式自查，不声称已通过外部查重平台。
- 答辩 PPT 初稿需要全组一起审核，本目录仅提供初稿和录制脚本。
