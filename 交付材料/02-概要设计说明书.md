## 1. 总体架构

系统采用前后端分离的教学实践结构。后端是单体 Spring Boot 应用，负责认证、业务规则、数据访问和接口输出；数据库使用 MySQL 保存角色、房间、预订、入住、账单、清洁和报表基础数据；前端由 uni-app 网页端和 APP 端源码组成，负责页面展示与接口调用。

## 2. 技术选型

| 层次 | 技术 | 说明 |
| --- | --- | --- |
| 后端 | Spring Boot 3.3.1 | Web、Validation、WebSocket、统一异常和鉴权 |
| 数据访问 | MyBatis | Mapper 注解和 XML 混合，实体来自 MySQL |
| 数据库 | MySQL 8.0 | `schema.sql` 建表，`seed.sql` 初始化测试数据 |
| 接口文档 | Knife4j / Swagger | `/swagger-ui.html` 和 `/doc.html` |
| 前端 | uni-app / Vue 单文件组件 | 网页端和 APP 端分别维护页面与资源 |

## 3. 后端模块划分

| 模块 | 责任 |
| --- | --- |
| `common` | 统一响应、异常处理、鉴权拦截、角色校验、业务异常 |
| `config` | Web MVC 拦截器和 WebSocket 配置 |
| `controller` | 接收 HTTP 请求，完成参数校验和响应封装 |
| `service` / `service.impl` | 业务流程编排、状态流转和金额计算 |
| `mapper` | MyBatis 数据访问接口 |
| `entity` | 数据库表实体 |
| `dto.request` / `dto.response` | 前后端传输对象 |
| `utils` | Token、金额、状态映射、ID 生成、房态推送 |

## 4. 核心业务流程

1. 登录：校验用户名和密码，生成 token，返回用户角色。
2. 新增预订：校验入住退房日期、房型和空房，创建预订并更新房态。
3. 入住：根据预订或 Walk-in 请求创建客史、入住记录和账单。
4. 消费入账：向账单追加服务消费，用 `BigDecimal` 重算金额。
5. 退房：结算账单，更新入住记录，房间转为脏房并创建清洁任务。
6. 清洁：任务状态流转，检查完成后房间恢复空净。
7. 报表：按数据库当前业务状态生成入住率和营收汇总。

## 5. 部署视图

| 节点 | 端口 | 内容 |
| --- | --- | --- |
| 后端服务 | `8080`，测试时可改 `18080` | Spring Boot API、Swagger、WebSocket |
| MySQL | `3306` | `hrms` 数据库 |
| 前端网页 | `8081`，Nginx 示例 | HBuilderX 发行后的静态资源 |

## 6. 设计约束

- 后端默认配置中包含本地数据库密码和 token secret，课程演示可用，生产环境需要环境变量或 profile 覆盖。
- 前端网页端 `utils/api.js` 当前写死 `http://localhost:8080`，部署时需要统一配置方式。
- APP 端 `pages.json` 含 JSONC 注释，HBuilderX 通常可识别，严格 JSON 校验工具会报错。
