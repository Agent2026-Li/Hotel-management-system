## 1. 数据库概况

数据库名为 `hrms`，由 `后端/hrms-backend/src/main/resources/db/schema.sql` 创建结构，并由 `seed.sql` 初始化演示与测试数据。测试完成后已重新导入脚本，将本机数据恢复到种子状态。

## 2. 核心表

| 表名 | 职责 | 关键数据 |
| --- | --- | --- |
| `sys_user` | 系统用户和登录账号 | 初始化 5 个测试账号 |
| `sys_role` | 角色信息 | 管理员、前台、客房等角色 |
| `room_type` | 房型定义 | 房型编号、名称、价格 |
| `room` | 房间和房态 | 初始化 98 间房 |
| `reservation` | 预订记录 | 初始化 2 条预订 |
| `guest` | 客户档案 | 住客基础信息 |
| `check_in_record` | 入住记录 | 入住、退房和房间占用记录 |
| `bill` | 账单主表 | 初始化 2 条账单 |
| `bill_item` | 账单明细 | 房费、服务费等 |
| `housekeeping_task` | 清洁任务 | 初始化 2 条任务 |

## 3. 数据关系

- 一个房型可对应多间房间。
- 一条预订关联房型、房间和住客信息。
- 一次入住生成一条入住记录、一个住客档案和一张账单。
- 一张账单包含多条账单明细。
- 退房后自动生成清洁任务，清洁完成后更新房态。

## 4. 测试数据量

| 表 | 记录数 |

| `bill` | 2 |
| `housekeeping_task` | 2 |

## 5. 设计注意事项

- 金额字段应使用精确小数类型，并在 Java 侧使用 `BigDecimal`。
- 房态、预订状态、清洁状态需要保持枚举一致。
- 测试前后建议重新导入 `schema.sql` 和 `seed.sql`，避免冒烟测试修改影响后续演示。
