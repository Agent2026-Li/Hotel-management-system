# 后端说明文档

## 架构

后端为单体 Spring Boot 工程，包名 `com.hrms`，采用 `controller / service / mapper / entity / dto / common / config / utils` 分层结构。当前实现使用原生 MyBatis 连接 MySQL，已移除 `DemoDataStore` 内存数据仓库。

```text
com.hrms
├─ config      配置类，例如 MVC 拦截器和 WebSocket
├─ controller  REST API 入口
├─ dto         前端请求和响应对象
├─ entity      数据库表实体
├─ mapper      MyBatis 数据访问接口
├─ service     业务接口
├─ service.impl 业务实现
├─ common      统一响应、异常、鉴权和权限注解
└─ utils       Token、状态转换、金额和 ID 工具
```

## Mapper 规则

- 简短 SQL 直接写在 `src/main/java/com/hrms/mapper/*Mapper.java`，例如按主键查询、单表新增、状态更新。
- 复杂 SQL 写在 `src/main/resources/mapper/*.xml`，例如房间列表关联房型和住客、预订动态查询、账单明细聚合、报表统计。
- 启动类使用 `@MapperScan("com.hrms.mapper")`，`application.yml` 配置 `mapper-locations: classpath:mapper/*.xml`。

## 核心业务规则

- 登录：`POST /api/auth/login`，账号来自 `sys_user`，角色来自 `sys_role`。
- 房态内部码：`VC` 空净房、`OC` 入住、`RS` 预订、`VD` 脏房、`CLEANING` 清洁中、`OOO` 维修。
- 房态接口返回：`vacant/occupied/reserved/dirty/cleaning/maintenance`。
- 预订：新增预订会锁定空净房为 `RS`；取消预订会把未入住房间恢复为 `VC`。
- 入住：Walk-in 和预订入住都会创建客史、入住记录、账单和房费明细，并把房间改为 `OC`。
- 退房：结清账单后入住记录改为 `checked_out`，房间改为 `VD`，并自动生成清洁任务。
- 清洁完成或检查通过：清洁任务改为 `done`，房间恢复 `VC`。
- 金额：所有账单金额统一使用 `BigDecimal`，账单总额 = 房费 + 服务消费，余额 = 总额 - 已付金额。

## 限制说明

真实 OTA、支付网关、OCR、短信、Redis 不在本次课程后端落地范围，当前只保留可联调的模拟字段和业务记录。
