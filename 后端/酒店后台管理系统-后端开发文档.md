## 1. 项目概览

本后端是酒店后台管理系统的单体 Spring Boot 服务，负责登录鉴权、RBAC 接口权限、房态、预订、入住、退房结算、账单、清洁、客史和报表等业务能力。

技术栈：

- Java 17
- Spring Boot 3.3.1
- Spring Web
- Spring Validation
- Spring WebSocket
- MyBatis 3.0.3
- MySQL 8
- Knife4j / Springdoc OpenAPI
- Maven

默认服务地址：

```text
http://localhost:8080
```

接口文档地址：

```text
http://localhost:8080/doc.html
http://localhost:8080/swagger-ui.html
```

## 4. 统一响应和鉴权

### 4.1 统一响应

所有接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

约定：

- `code = 0` 表示成功。
- `code != 0` 表示失败。
- 鉴权失败仍返回 HTTP 200，但业务 `code` 为 `401` 或 `403`。

### 4.2 登录 Token

登录接口：

```http
POST /api/auth/login
Content-Type: application/json
```

请求：

```json
{
  "username": "admin",
  "password": "123456"
}
```

成功后返回 `token/tokenType/username/name/role/expiresInHours`。

除以下接口外，其他 `/api/**` 接口都需要请求头：

```http
Authorization: Bearer <token>
```

免登录接口：

- `POST /api/auth/login`
- `GET /api/auth/test-accounts`

### 4.3 Token 实现

Token 由 `com.hrms.utils.TokenService` 生成，格式为：

```text
base64Url(username|name|role|expiresAt).HmacSHA256签名
```

有效期由配置控制：

```yaml
hrms:
  auth:
    secret: hrms-demo-secret-change-in-production
    token-valid-hours: 8
```

生产环境必须修改 `hrms.auth.secret`。

## 7. 核心业务流程

### 7.1 新增预订

接口：

```http
POST /api/reservations
```

主要流程：

1. 校验入住日期、离店日期、房型。
2. 如果未传 `roomNumber`，系统自动选择对应房型的空净房。
3. 计算入住晚数和金额。
4. 创建预订记录。
5. 将对应房间状态更新为 `RS`。

请求示例：

```json
{
  "name": "测试预订",
  "phone": "13800001111",
  "roomType": "RT002",
  "checkin": "2026-07-06",
  "checkout": "2026-07-08",
  "remark": "联调测试"
}
```

### 7.2 编辑预订

接口：

```http
PATCH /api/reservations/{id}
```

规则：

- 可修改客户姓名、手机号、房型、房号、入住日期、离店日期、备注。
- 会重新计算晚数和金额。
- 如果变更房间，应同步维护旧房间和新房间状态。

### 7.3 取消预订

接口：

```http
POST /api/reservations/{id}/cancel
```

规则：

- 只有未入住的预订允许取消。
- 取消后预订状态为 `cancelled`。
- 如果该预订锁定了房间，房间恢复为 `VC`。

### 7.4 办理入住

接口：

```http
POST /api/checkin
```

支持两种入住方式。

预订入住：

```json
{
  "reservationId": "R20260701001"
}
```

Walk-in 入住：

```json
{
  "roomNumber": "106",
  "name": "测试入住",
  "phone": "13800009999",
  "idCard": "370100200001010099",
  "checkoutDate": "2026-07-03",
  "deposit": 300
}
```

主要流程：

1. 创建或更新客户档案。
2. 创建入住记录。
3. 创建账单主表。
4. 创建房费账单明细。
5. 房间状态更新为 `OC`。
6. 预订入住时，预订状态更新为 `checked_in`，并记录实际入住时间。

### 7.5 退房结算

接口：

```http
POST /api/checkout
```

请求示例：

```json
{
  "roomNumber": "101",
  "paymentMethod": "cash",
  "paidAmount": 398
}
```

规则：

- 可以通过 `roomNumber` 或 `checkInId` 找到在住记录。
- 结清账单后，账单状态改为 `paid`。
- 入住记录状态改为 `checked_out`。
- 房间状态改为 `VD`。
- 自动生成退房清洁任务。

### 7.6 消费入账

接口：

```http
POST /api/bills/{id}/charges
```

请求示例：

```json
{
  "itemName": "迷你吧",
  "category": "MINIBAR",
  "amount": 25.00,
  "quantity": 1,
  "remark": "饮料"
}
```

规则：

- 新增 `bill_item`。
- 重新统计 `room_amount`、`service_amount`、`total_amount`。
- 金额统一使用 `BigDecimal`。

### 7.7 清洁任务

查询：

```http
GET /api/housekeeping/tasks?status=pending&assignedTo=housekeeper
```

更新：

```http
PATCH /api/housekeeping/tasks/{id}
```

请求示例：

```json
{
  "status": "inspected",
  "remark": "检查通过"
}
```

规则：

- 清洁员可以查看和更新清洁任务。
- 客房经理可以分配、检查任务。
- 清洁完成或检查通过后，房间恢复 `VC`。

## 13. 开发约定

### 13.1 Controller

- Controller 只负责参数接收、权限注解和调用 Service。
- 需要控制角色访问时添加 `@RoleRequired`。
- 返回值统一使用 `ApiResponse.success(data)`。

### 13.2 Service

- 业务规则写在 `service.impl`。
- 涉及多表写入的业务需要加事务。
- 抛业务异常时使用 `BusinessException`，不要直接返回字符串错误。

### 13.3 Mapper

- 简单 SQL 可以写在 Mapper 接口注解里。
- 复杂 SQL 写在 `src/main/resources/mapper/*.xml`。
- 注意开启了驼峰映射：

```yaml
mybatis:
  configuration:
    map-underscore-to-camel-case: true
```

### 13.4 金额

- 金额一律使用 `BigDecimal`。
- 账单金额由 `AmountUtils` 统一计算。
- 不要在金额字段上使用 `double` 或 `float`。

### 13.5 状态

- 房态入库使用内部码。
- 前端接口返回前端值。
- 状态转换统一走 `StatusMapper`。

## 14. 常见问题

### 14.1 登录提示无权限

检查：

- 请求头是否带了 `Authorization: Bearer <token>`。
- 当前用户角色是否在接口 `@RoleRequired` 列表中。
- Token 是否过期，默认 8 小时。

### 14.2 数据库连接失败

检查：

- MySQL 是否启动。
- 是否存在 `hrms` 数据库。
- 是否执行了 `schema.sql` 和 `seed.sql`。
- `application.yml` 中用户名和密码是否正确。

### 14.3 端口被占用

默认端口是 8080。可以关闭旧进程，或临时修改：

```yaml
server:
  port: 8082
```

### 14.4 前端接口跨域

`WebConfig` 已配置跨域和鉴权拦截器。如果新增接口在 `/api/**` 下，默认会经过鉴权。
