# 部署手册

## 环境

- JDK 17 或更高
- Maven 3.6+
- MySQL 8.0

## 数据库

在 MySQL 客户端或数据库工具中按顺序执行：

```sql
source 后端/hrms-backend/src/main/resources/db/schema.sql;
source 后端/hrms-backend/src/main/resources/db/seed.sql;
```

如果使用图形化工具，先运行 `schema.sql`，再运行 `seed.sql`。脚本会创建 `hrms` 数据库、98 间房、角色账号、预订、在住、账单和清洁任务测试数据。

## 配置

数据库连接在 `后端/hrms-backend/src/main/resources/application.yml`：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/hrms?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai
    username: root
    password: 212324
```

如本机密码不同，只修改这里。

## 启动

推荐在 IDEA 中直接运行 `com.hrms.HrmsBackendApplication`。

命令行启动：

```powershell
cd "C:\Users\Trump\Desktop\Hotel-management-system\后端\hrms-backend"
$env:JAVA_HOME='D:\develop\Java\Jdk-21'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
mvn test
mvn package -DskipTests
java -jar target\hrms-backend-0.1.0.jar
```

说明：当前路径包含中文时，`mvn spring-boot:run` 可能出现启动类加载失败；IDEA 或 `java -jar` 启动正常。

默认端口：`8080`。

## 文档地址

- Knife4j / Swagger：`http://localhost:8080/doc.html` 或 `http://localhost:8080/swagger-ui.html`
- 测试账号接口：`http://localhost:8080/api/auth/test-accounts`
- HTTP 样例：`后端/http/hrms-api.http`

## 前端页面打开

网页端前端是 HBuilderX/uni-app 项目。先用 HBuilderX 打开 `前端-app和网页/前端-网页版本-酒店系统`，通过 `发行 -> 网站-PC Web或手机H5` 生成 `unpackage/dist/build/web`，再复制到 `nginx-1.18.0/html/hrms-web`。

Nginx 配置模板见 `后端/nginx/hrms-web.conf`。配置完成后访问：

```text
http://localhost:8081
```

## 常见问题

- 访问接口提示数据库错误：先确认 MySQL 已启动，并已按顺序导入 `schema.sql` 和 `seed.sql`。
- 登录失败：确认 `sys_user` 表中有 `admin / 123456` 等测试账号。
- 8080 被占用：先关闭旧后端进程，或临时修改 `application.yml` 的 `server.port`。
