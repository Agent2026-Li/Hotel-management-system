### 4.4 数据库连通性与初始化

首次使用仓库默认密码连接时失败：

```text
jdbc:mysql://localhost:3306/hrms?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&connectTimeout=3000
username=root
password=212324
```

结果：

```text
java.sql.SQLException: Access denied for user 'root'@'localhost' (using password: YES)
```

结论：

当前环境 MySQL 服务存在并运行，但 `application.yml` 中的默认数据库密码不可用。用户提供有效本机密码后，已通过启动参数覆盖数据库密码，并成功执行 `schema.sql`、`seed.sql` 初始化 `hrms` 测试库。

## 5. 缺陷与风险

### D-001 默认数据库凭据不可用

| 项目 | 内容 |
| --- | --- |
| 严重级别 | 中 |
| 类型 | 配置风险 |
| 现象 | 使用仓库默认 `application.yml` 启动时，依赖数据库的接口会因 MySQL 认证失败返回 `code=500` |
| 根因 | `application.yml` 中默认数据库密码与本机 MySQL root 密码不一致 |
| 影响 | 新环境无法直接运行后端；必须手工覆盖数据库密码 |
| 当前处理 | 本次复测通过启动参数覆盖数据库密码后，业务接口已验证通过 |
| 建议 | 建立 `dev/test` profile 或使用环境变量覆盖数据库连接信息；补充数据库初始化说明 |

### R-001 敏感配置硬编码

| 项目 | 内容 |
| --- | --- |
| 严重级别 | 中 |
| 位置 | `后端/hrms-backend/src/main/resources/application.yml` |
| 现象 | 数据库密码、Token secret 明文写入仓库 |
| 影响 | 不适合生产或公开环境；也会导致不同机器上无法直接运行 |
| 建议 | 使用环境变量或 profile 覆盖，例如 `${DB_PASSWORD}`、`${HRMS_AUTH_SECRET}` |

### R-002 前端后端地址硬编码

| 项目 | 内容 |
| --- | --- |
| 严重级别 | 中 |
| 位置 | `前端-app和网页/前端-网页版本-酒店系统/utils/api.js` |
| 现象 | `BASE_URL = 'http://localhost:8080'` |
| 影响 | 本机 8080 被占用时无法切换；部署到测试/生产环境需要改源码 |
| 建议 | 使用环境配置、代理路径 `/api` 或 uni-app 条件编译配置 |

### R-003 后端全局异常响应信息不足

| 项目 | 内容 |
| --- | --- |
| 严重级别 | 中 |
| 现象 | 数据库异常时接口返回 `{"code":500,"message":null,"data":null}` |
| 影响 | 前端和测试人员难以定位失败原因；日志中也未直接打印完整异常 |
| 建议 | 全局异常处理记录服务端日志，并给客户端返回稳定的非空错误消息，例如 `系统繁忙，请稍后重试` |

### R-004 前端工程缺少 CLI 构建入口

| 项目 | 内容 |
| --- | --- |
| 严重级别 | 中 |
| 现象 | 两个前端目录均未提供 `package.json` |
| 影响 | 无法在 CI 中执行 `install/build/test`；测试依赖 HBuilderX 手工操作 |
| 建议 | 补充工程化入口，至少提供 `package.json`、构建命令和 README 运行说明 |

### R-005 调试日志保留

| 项目 | 内容 |
| --- | --- |
| 严重级别 | 低 |
| 位置 | 两个前端工程的 `App.vue` |
| 现象 | 存在 `console.log('App Launch')`、`console.log('App Show')`、`console.log('App Hide')` |
| 影响 | 生产环境控制台噪声 |
| 建议 | 使用条件编译或日志封装，仅在开发环境输出 |
